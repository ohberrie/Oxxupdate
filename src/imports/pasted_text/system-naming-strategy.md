1️⃣ Red Team 평가 — 현재 네이밍 상태

❌ 위험 포인트

1. 개별 이름은 괜찮은데 “시스템”처럼 안 느껴질 수 있음

예:
	•	Dense Field
	•	Modular Fill
	•	Layer Stack
	•	Selection Logic

→ 각각은 좋다
→ 하지만 리스트로 보면 아직 약간:

디자인 개념 모음 느낌

이지,

operating system 느낌

까지는 아니다.

⸻

2. Level mismatch 가능성 (아직 남아 있음)
	•	Dense Field → spatial phenomenon
	•	Modular Fill → generation method
	•	Layer Stack → structure
	•	Selection Logic → decision system

👉 이건 사실 의도적으로 다른 레벨이라 괜찮다.
하지만 그 대신 반드시 필요하다:

subtitle로 레벨을 명확히 구분

⸻

3. 기억성 vs 신뢰성 밸런스
	•	너무 시적 → 가벼워짐
	•	너무 기술적 → 매력 떨어짐

현재는 중간인데,
약간 더 “정제된 시스템 느낌”으로 밀어야 한다.

5️⃣ Dense Field 페이지

Overview

Dense Field defines how visual intensity is distributed across space.
It determines where attention accumulates and how spatial tension is perceived.

⸻

Principle
	•	uniform distribution
	•	gradient distribution
	•	clustered density
	•	void vs concentration

••• ••• •••

👉 다이어그램 키워드:
	•	uniform
	•	gradient
	•	cluster
	•	void

⸻

Experiment
	•	voxel density variations
	•	lighting concentration tests
	•	stage attention mapping

⸻

	•	clustered density → focal point 생성
	•	gradient density → movement 유도
	•	uniform density → 안정감

	•	Stage visual focus design

6️⃣ Modular Fill 페이지

Overview

Modular Fill generates complex forms by aggregating simple repeated units.

⸻

Principle
	•	unit repetition
	•	rule-based filling
	•	inside-out filling logic
	•	boundary-driven growth

⸻

Diagram 구조

Step 1: Boundary

[     curve      ]

Step 2: large modules

[■■■■■■]

Step 3: smaller modules fill gaps

[■■■■■■■■■■■■]

👉 키워드:
	•	fill order
	•	module scale hierarchy
	•	gap resolution

⸻

Experiment
	•	grid fill vs adaptive fill
	•	voxel vs custom module
	•	density-aware filling

⸻

Observation
	•	큰 모듈 → 구조 형성
	•	작은 모듈 → 디테일 채움
	•	fill 순서 → 형태의 character 결정

⸻

Applications
	•	Shaper system
	•	Pixel Grid Structures
	•	Stage modular installations

8️⃣ Selection Logic 페이지

Overview

Selection Logic determines how a system chooses one outcome among many possible configurations.

⸻

Principle
	•	constraint filtering
	•	weighted selection
	•	seed variation
	•	rule-based pruning

⸻

Diagram 구조

All Possibilities
[ A B C D E F G ]

↓ filter

[ B D F ]

↓ selection

[ D ]

또는

Parameters → Variations → Selection

[ sliders ]
     ↓
[ 100 forms ]
     ↓
[ 1 chosen ]


⸻

Experiment
	•	random vs constrained selection
	•	parameter sweeps
	•	preset generation

⸻

Observation
	•	unconstrained → noise
	•	constraints → clarity
	•	good selection = identity

⸻

Applications
	•	Shaper configurator (핵심)
	•	preset system
	•	generative design workflows

9️⃣ Red Team 최종 평가

지금 이 구조가 완성되면:

사용자는 이렇게 이해한다:

OXX는 디자인 스타일이 아니라

1. 공간을 정의하는 방식 (Dense Field)
2. 형태를 만드는 방식 (Modular Fill)
3. 구조를 조직하는 방식 (Layer Stack)
4. 결과를 선택하는 방식 (Selection Logic)

→ 이 4개로 작동하는 시스템이다
